## Observation
My plan treated the Node version as a non-blocking Consideration ("confirm local Node satisfies Vite"), leaving the runtime version unpinned. The reviewer promoted it to a hard requirement: pin Node to the latest current LTS and manage it with Volta (voltajs), so package.json carries an exact volta.node version and the correct Node auto-loads for the app. The underlying want is a reproducible, pinned toolchain — the same determinism motive behind the earlier exact-dependency-version rule. This is a standing project convention (pin the runtime, not just libraries), so every future scaffolding/toolchain change should assume Volta-pinned Node rather than relying on whatever Node the machine happens to have.
