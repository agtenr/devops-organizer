## Observation
The pass-1 review's top finding was that a required test file (ConfirmDeleteDialog.test.tsx) existed on disk and passed locally but had never been git-added, so it was absent from the PR. The coder's self-check "npm run test passes" gave false confidence because Vitest runs every test file on disk regardless of git tracking — an uncommitted test still turns the suite green locally, so the gap is invisible to the runner and only surfaces in the PR diff. This is a recurring failure class for file-granular commits (adding files by explicit path and forgetting the sibling test): the green-suite signal does not verify that the tests are actually included in the change under review.
