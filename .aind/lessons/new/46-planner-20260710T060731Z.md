## Observation
Story 46 requires Playwright validation of whole-app layout behaviors (only-the-list-scrolls, fixed tabs/filters, the loading state that hides tabs/filters). But the app is MSAL-auth-gated and the only existing E2E harness (harness.tsx / harness.html) mounts EmailList in isolation with mock props — there is no established pattern for driving the full app shell (App/Organizer/TopBar) in a browser without real auth. This forced the plan to invent a per-story seam (an injected data hook on Organizer plus a static header stand-in) just to make the shell testable. The gap: testing.md documents the EmailList harness but gives no convention for E2E-validating auth-gated, whole-layout acceptance, so each layout/shell story has to re-solve "how do I drive the real app under Playwright" from scratch.
