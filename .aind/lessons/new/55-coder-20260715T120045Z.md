## Observation
The Playwright harness E2E suite is timing-fragile on a cold checkout: the first navigation triggers a one-time Vite dependency pre-bundle (~25s), and because tests run in parallel the entire first wave blocks on it, so each first-wave test clocks ~28s against Playwright's default 30s timeout while later (warm) tests finish in ~3-4s. A newly added interaction test (a drag) that adds even ~2s of its own work on top of the cold preamble intermittently exceeds 30s and fails, then passes cleanly warm or single-worker. The cause is cold dep-optimization contention under parallel workers, not test logic — it means any harness test whose own work is non-trivial sits close to the timeout ceiling on a cold run.
