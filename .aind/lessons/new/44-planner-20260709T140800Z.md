## Observation
The categorization taxonomy in the model (Build/Pull-request sub-type unions) and the classify() rule set are anchored to "story 37 §3's reference table" as the authoritative source, but the demo corpus itself contains ADO notification types absent from that table — "Pull request published" and "Build cancelled" — which is why they mis-categorized. The cause is that the reference table treated as the taxonomy source is not exhaustive of the message types actually present in the sample .msg corpus the rules are supposed to cover, so new stories keep discovering real types the taxonomy never enumerated.
