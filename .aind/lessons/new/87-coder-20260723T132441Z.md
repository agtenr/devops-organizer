## Observation
A mutable mock state variable (let mockThemeMode) lets per-test code override a hoisted vi.mock — the factory hoists but the getter runs at call time. This pattern avoids the "dead branch" anti-pattern where renderTopBar accepts a param that never takes effect because the mock is fixed at module level. Useful for mocking context providers in component tests where vi.doMock is complex.
