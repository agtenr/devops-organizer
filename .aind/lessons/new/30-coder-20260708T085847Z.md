## Observation
On Windows, `npm run format:check` (prettier --check) fails on nearly every pre-existing file because git checks them out with CRLF (core.autocrlf) while Prettier expects LF — even though the committed git objects are LF and clean, and `main` passes. This produces a misleading red format:check locally that has nothing to do with the current change, forcing the coder to prove cleanliness via `git show <file> | prettier` on the normalized objects instead of trusting the working-tree check. Cause: the repo has no .gitattributes enforcing `eol=lf` and Prettier is not set to `endOfLine: auto`, so line endings diverge between the working tree and the committed content on CRLF platforms.
