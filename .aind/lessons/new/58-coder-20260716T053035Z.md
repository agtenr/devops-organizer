## Observation
Second human correction on the same screenshot-link issue, refining my earlier lesson which was itself incomplete. An absolute GitHub *blob* URL (https://github.com/<owner>/<repo>/blob/<branch>/<path>.png) embedded as a markdown image in a PR description does NOT render inline — GitHub serves the blob HTML page, not the image bytes, so the image shows broken. To render inline, the URL must point at the raw bytes: either append the query param `?raw=true` to the blob URL (https://.../blob/<branch>/<path>.png?raw=true) or use the raw path form (https://.../raw/<branch>/<path>.png), or raw.githubusercontent.com. Net: for a committed screenshot referenced from a PR description, a repo-relative path breaks (resolves against the PR URL) AND a plain blob URL breaks (renders the file page, not the image); only a raw-bytes URL renders. This recurs on every screenshot-bearing PR and the testing rule's wording ("a relative repo path or an inlined image") gives no hint of it.
