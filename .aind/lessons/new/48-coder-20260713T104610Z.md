## Observation
The first component test that used a Fluent UI MessageBar crashed in jsdom with "ResizeObserver is not a constructor". Fluent's MessageBar attaches a ResizeObserver on mount (useMessageBarReflow), and the shared jsdom setup (src/setupTests.ts) provided no ResizeObserver polyfill. The testing rule tells authors to render Fluent components through the FluentProvider wrapper but says nothing about the jsdom APIs (ResizeObserver, and likely matchMedia/IntersectionObserver) that layout-aware Fluent v9 components require — so the gap only surfaces the first time such a component is tested. It will recur for any future Fluent component that observes size/layout.
