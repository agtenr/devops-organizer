## Observation
For the categorization fixtures I assumed the coder would capture raw Message JSON from the live MailDebug dump, because the repo had no visible sample corpus and neither testing.md nor categorization-domain.md points to one. The reviewer corrected this: a real sample corpus already exists as 14 Outlook .msg files under /design/demo-messages, one per row of the story's reference-examples table, and those are the intended source for the categorization unit tests. This matters because testing.md mandates fixture-driven tests "captured as real sample e-mails" but never names where that corpus lives, and the corpus itself is currently untracked and in a binary (.msg) format that the test suite can't read directly — so a planner cannot discover it and has to invent a fixture-capture path. The cause is a gap between the testing rule's stated fixture practice and the absence of any documented, tracked, machine-readable fixture location for it.
