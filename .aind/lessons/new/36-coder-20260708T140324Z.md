## Observation
On this Windows checkout `npm run format:check` (the lint skill's Prettier gate) reports every one of the 21 tracked files as unformatted — including files untouched by the change (e.g. TopBar.tsx from a prior story). Cause: the repo has no `.gitattributes` and Prettier's config sets no `endOfLine`, so files stored LF in the git index are checked out CRLF on Windows (autocrlf), and Prettier's default `endOfLine: lf` flags the CRLF working tree. The committed (LF) content is actually fine and Prettier passes on a Linux/CI checkout, but a coder on Windows cannot trust `format:check` as a signal and risks a repo-wide reformat that is pure line-ending churn. The frontend-architecture rule mandates Prettier but the toolchain lacks the line-ending pin that makes it deterministic across OSes.
