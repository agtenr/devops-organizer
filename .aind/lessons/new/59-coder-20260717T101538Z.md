## Observation
The AIND worktree tooling repeatedly blocked `git rebase` in the implement phase. The worktree config's copyFiles list includes the folder src/services/categorization/__fixtures__, which contains a git-TRACKED file (README.md). Every invocation of aind-revise-code-pr.sh (including its "rebase" subcommand) re-copies that folder into the worktree; on Windows the copied README.md is renormalized CRLF->LF and shows as an unstaged modification, so `git rebase` aborts with "cannot rebase: You have unstaged changes" before it even starts. The script therefore could not perform the rebase itself — I had to `git checkout --` the fixtures file and run `git fetch` + `git rebase origin/main` manually. The cause is that copyFiles copies a directory containing a version-controlled file rather than only gitignored artifacts, and the copy dirties the tree right before a git operation that requires a clean tree.
