## Observation
The human accepted the reviewer's suggestion to add a `format:check` (`prettier --check .`) script, because the frontend-architecture "done" bar and the DoD both assert "Prettier reports no formatting changes needed" yet nothing in package.json enforced it — `lint` ran only eslint and `format` only rewrote files. The human wants formatting compliance verifiable by a single command, so a `format:check` script (and treating it as the canonical way to assert formatting in the DoD/CI) is the project's preferred convention going forward.
