## Observation
The `lint` script runs only `eslint .`; Prettier is wired as `format` (`prettier --write`) with no `--check` variant, so the frontend-architecture "done" bar and the plan DoD item "Prettier reports no formatting changes needed" cannot be verified mechanically by a single command — it relies on a human/agent running `prettier --check` by hand. This is a standing gap that will recur on every story, since formatting compliance is asserted in the rules but not enforced by any runnable check in package.json.
