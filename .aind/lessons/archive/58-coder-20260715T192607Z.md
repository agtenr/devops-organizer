## Observation
The human corrected the code PR: the committed screenshot was embedded in the PR description with a repo-relative markdown image path (![alt](e2e/screenshots/58/selected-filters.png)). Relative links do NOT resolve in a GitHub PR *description* — GitHub resolves them against the PR URL, producing a broken link like https://github.com/<owner>/<repo>/pull/e2e/screenshots/58/... (it renders/links correctly only inside committed README/markdown files in the repo, not in the PR body). The fix is an absolute URL to the file on the branch (e.g. https://github.com/<owner>/<repo>/blob/<branch>/<path>). Cause: the testing rule ("The code PR references every screenshot — embed or link each committed screenshot (a relative repo path or an inlined image)") reads as if a relative repo path works in the PR description, but a relative path only works in-repo, not in the PR description, so following it literally yields a broken image link every time a UI story attaches its screenshot. Recurs on every screenshot-bearing PR.
