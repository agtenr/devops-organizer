## Observation
The authentication.md "Scope staging" note frames the mail-folder read as an open design choice — "folder by name vs. resolving a folder id first." Planning showed this isn't actually a free choice for this app: the target folder "DevOps" is a custom (non-well-known) Outlook folder, and Graph's well-known-name shortcut (/me/mailFolders('DevOps')) only resolves default folders, so resolving the id by displayName filter first is effectively mandatory. The rule presented as an open question something that a Graph domain fact dictates, so a planner has to rediscover the constraint from the API docs rather than the rule. Cause: the rule captured the decision point but not the domain fact (custom vs well-known folders) that constrains it.
