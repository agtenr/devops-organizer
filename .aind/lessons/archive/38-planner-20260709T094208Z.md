## Observation
The planner initially placed the tab-selection state and the fetch/categorize data inside the temporary MailDebug component (a throwaway visualizer slated for removal in #39), treating "MailDebug is temporary" as license to co-locate the transitional wiring there. The human rejected this: state that outlives the visualizer (selection state, the real data path) must live in a permanent container even during a transitional story, so it isn't orphaned when the debug component is deleted. Cause: the plan conflated "the display surface is temporary" with "the feature state wired to it is temporary."
