## Observation
Two build/test frictions hit while implementing inside the AIND git worktree (.claude/worktrees/58-impl): (1) the worktree is created without node_modules (the worktree helper copies aind.env/.env/fixtures/demo-messages but not the installed dependencies), so npm run build/test/lint cannot run there until node_modules is made available — I worked around it with a Windows directory junction to the main checkout's node_modules rather than a full reinstall. (2) With node_modules present, the project's `npm run test` (vitest run --no-isolate, default threads pool) crashed immediately with "Worker exited unexpectedly" / unhandled 'error' event and exited the process before running any test; switching to `vitest run --pool=forks` ran the full suite reliably. Cause is environmental (worktree isolation + the default vitest worker pool on this Windows/Node setup), but both recur on every worktree-based implement run here, so a coder without this context can lose time diagnosing a green-on-main suite that won't even start in the worktree.
