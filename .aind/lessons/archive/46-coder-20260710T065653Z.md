## Observation
Running the Playwright suite for story 46 surfaced TWO pre-existing e2e tests that had been silently broken by earlier stories, unrelated to this story's changes: (1) the harness columns test queried 'Date' as an ARIA columnheader, but Fluent's Table marks the first data column a rowheader — which flipped when story 43 added the leading selection checkbox column; (2) the smoke test asserted a hello-world heading that story 30 removed when it added the MSAL auth gate (its App.test.tsx sibling was updated then, but the e2e smoke test was not). Both went undetected because e2e is not wired into CI (CLAUDE.md notes no CI/CD is configured), so nobody runs `npm run test:e2e` between stories. The cause: without e2e in CI, browser tests rot across stories and only fail when a much later story happens to run the suite, forcing that story to absorb unrelated fixes.
