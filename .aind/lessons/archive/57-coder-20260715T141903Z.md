## Observation
In worktree-based implement runs, a fresh implement worktree starts with no node_modules AND without the categorization suite's fixtures: testing.md correctly says the __fixtures__/*.json are gitignored and generated locally, but they only exist in whatever checkout generated them (the main checkout), not in a sibling git worktree. So `npm run test` cannot pass in a fresh worktree until you both `npm ci` and manually copy src/services/categorization/__fixtures__/*.json from the main checkout. The friction is that the worktree flow gives every implement run an empty, fixture-less tree, but the "regenerate fixtures locally" step assumes the raw .msg corpus is present — which it also isn't in the worktree.
