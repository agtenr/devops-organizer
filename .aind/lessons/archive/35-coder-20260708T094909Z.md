## Observation
On Windows the repo's `npm run format:check` (Prettier) fails on ALL ~20 tracked source files, not just the changed ones. Cause: git stores content as LF but checks it out CRLF (autocrlf) while Prettier's default endOfLine is "lf", so every working-tree file reads as CRLF and is flagged; the git-stored blobs are actually Prettier-clean. The repo has no `.gitattributes` and no `endOfLine` setting in `.prettierrc`, so there is nothing normalizing line endings. Every Windows contributor and every implement run hits this false-positive format failure, which obscures whether a change is genuinely mis-formatted.
