## Observation
The review flagged (as a nit) that src/components/SidebarFilters/useSidebarFilters.ts uses the `use`-prefix naming convention but exports only pure functions (typeKey, deriveProjectOptions, filterByProject, …) and no React hook. This is the second occurrence of the same shape — CustomerTabs/useCustomerTabs.ts likewise exports the pure deriveCustomerTabs alongside its hook, and here there is no hook at all. Cause: frontend-architecture.md prescribes "logic lives in a colocated custom hook (e.g. useCustomerTabs.ts)" but the codebase has settled into putting pure, React-free derivation helpers in that same `use*`-named file; the rule doesn't say where pure (non-hook) logic belongs, so a `use*` filename ends up carrying zero hooks. Recurs because the rule conflates "the colocated logic file" with "a custom hook".
