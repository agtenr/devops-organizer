## Observation
On the package-scope thread the reviewer accepted installing dependencies as-we-go but corrected how they must be declared: every dependency in package.json must be pinned to an exact version — no ^, ~, or * ranges (install via npm --save-exact / .npmrc save-exact=true). My plan had not specified a versioning policy, so it would have inherited npm's default caret ranges. The reviewer wants deterministic, reproducible installs for this project; this is a standing dependency-declaration convention, not a one-off, so it applies to every future story that adds or bumps a package.
