## Observation
Fluent v9 DataGrid's `resizableColumns` defaults `resizableColumnsOptions.autoFitColumns` to true, which re-fits every column to the container width on each render. This silently undermined two plan goals at once: it shrank columns below their configured idealWidth (Subject rendered 328px against an ideal of 360), and it immediately re-fit after a manual resize so drags did not stick (both mouse-drag and keyboard resize left the width unchanged in the Playwright E2E). Neither the plan nor the Fluent type docs flagged this default; it only surfaced when the resize E2E failed. Fix was `resizableColumnsOptions={{ autoFitColumns: false }}`, after which resizes persist and per-column ideal/compact widths hold. Reusable signal for future DataGrid work in this project: enabling resizableColumns without disabling autoFitColumns will make column-width assertions and manual resizing behave unexpectedly.
