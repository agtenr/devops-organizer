## Observation
The UI layout invariant in frontend-architecture.md states the top navigation bar shows "only the logged-in user's display name and a log-out button — nothing else," but story 35 (the top-bar story) requires the app title centred in that same bar. The rule was written as an absolute ("nothing else") before the story that defines the bar's actual contents existed, so a rule intended as guidance now reads as a hard contradiction of a legitimate story. Cause: an early architecture rule pinned an exhaustive "only X, nothing else" list for a component whose own defining story hadn't been planned yet.
