## Observation
The story specified the Microsoft Graph scope `Files.ReadWrite.AppFolder` for persisting data to OneDrive, but that scope is documented as valid only for personal Microsoft accounts — on OneDrive for Business (work/school) it is unsupported. This app's authentication.md establishes sign-in with organizational Entra ID accounts, so the story's named scope would not have worked as written. The signal: stories that name a specific Graph scope/API don't reconcile it against the app's account-type auth model (personal vs work/school), and authentication.md documents the account type but not which OneDrive/Files scopes are viable under it — so the mismatch only surfaces at plan time when the planner cross-checks the docs.
