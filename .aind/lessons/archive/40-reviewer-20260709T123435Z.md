## Observation
Cold review pass 1 returned CLEAN, but the human then found three live layout defects (invisible 0-width body panel on row click, evenly-split columns, wrapping badge). The reviewer re-grounds from artifacts only — the diff, the plan, the rules, and the coder's tests — and all of those were internally consistent: the jsdom component tests were present, green, and asserted the must-cover DOM behaviour. Nothing in the reviewable artifacts exercises rendered layout, so an approval based on reading code plus green jsdom tests structurally cannot catch a drawer that is open in the DOM but 0px wide, or columns with the wrong CSS widths. This recurs for any UI story whose acceptance is inherently visual: the cold-review gate has no live-rendering / screenshot step, so visual correctness is unverifiable from the artifacts it is given.
